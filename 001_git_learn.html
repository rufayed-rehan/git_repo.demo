GIT BASICS

>

WORK-FLOW :


Github repo creation =>> Clone link =>> Modify =>> Add =>> Commit =>> Push

clone :


cloning repo on our local machine
(remote machine (github) to local machine (our laptop,pc) copy)
=>> git clone {repo link}

change directory (folder) :


=>> cd {folder name}
~> to get out of the directory =>> cd ..

make directory (folder) :


makes a new folder
=>> mkdir {folder/repo name}

list :


shows the list of files in the folder
=>> ls

status :


display the state of the code
check if the file (in the code editor) is modified (edited from github) or not
=>> git status

untracked file :


new file (created on the code editor), that git does not yet track

staged (added) :


file is ready to be commited

add :


adds new/changed files on the working directory (folder) to git staging area
=>> git add {file name}
=>> to add all the modified (changed/edited) files together =>> git add .

commit :


the record of change
=>> git commit -m "some short message"

push :


uploads local repo content (edited or created files on the laptop/pc) to remote repository (github)
=>> git push origin main
origin=repo; main=branch name

init command :


to create ne git repo from the code editor
=>> git init ~>to make the repo a git repo
=>> git remote add origin {link} ~> git remote add; the origin will be set on the link
=>> git remote -v ~> to verify remote
=>> git branch ~> to check branch (like master,main...)
=>> git branch -M {new branch name} ~> to rename branch
=>> git push origin main
